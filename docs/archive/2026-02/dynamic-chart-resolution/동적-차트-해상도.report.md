## 3. PDCA 사이클 요약

### 3.1 Plan (계획) 단계

**목표**: 차트 확대 시 자동으로 더 세밀한 데이터 표시 (지도 앱처럼)

**핵심 요구사항**:
- 4단계 Zoom Level 시스템 (Level 0: 15분, Level 1: 1분, Level 2: 10초, Level 3: 1초)
- Zoom ratio에 따라 자동으로 데이터 간격(interval) 전환
- 레벨별 독립 캐싱 (중복 API 요청 방지)
- 500ms 디바운싱 (연속 확대 시 API 폭주 방지)
- 이전 레벨 데이터 유지 (API 실패 시)

**범위**:
- In Scope: 4단계 Zoom Level, 자동 interval 선택, SWR 캐싱, 디바운싱, 에러 처리
- Out of Scope: WebSocket 실시간 스트리밍, TimescaleDB 최적화, 차트 외 컴포넌트 수정

**성공 기준**:
- 4단계 Zoom Level 동작 정상
- 각 Level별 API interval 파라미터 전송 확인
- 캐시 적중률 90% 이상
- TypeScript 타입 에러 없음

### 3.2 Design (설계) 단계

**아키텍처**:
```
MON002LineDetail (페이지)
  ├─ initialInterval 선택 ("15분" / "1분")
  ├─ useDynamicResolution 훅 (커스텀)
  │  ├─ currentInterval 상태
  │  ├─ handleZoom(zoomRatio) 핸들러
  │  ├─ 500ms 디바운싱
  │  └─ fetchRangeData(SWR) 호출
  └─ TrendChart 컴포넌트
     ├─ uPlot setSelect 훅으로 zoom 감지
     ├─ onZoomChange 콜백 호출
     └─ 로딩 오버레이 표시
```

**API 스펙**:
```
GET /api/facilities/{facilityId}/power/range
Query:
  - startTime: ISO8601
  - endTime: ISO8601
  - interval: "15m" | "1m" | "10s" | "1s"
  - maxPoints: number (optional)

Response:
{
  "data": [ { "time": "HH:mm:ss", "power": number, "air": number }, ... ],
  "metadata": {
    "interval": "1m",
    "totalPoints": 480,
    "returnedPoints": 480,
    "downsampled": false,
    "zoomLevel": 1
  }
}
```

**Zoom Level ↔ Interval 매핑**:
| Level | 가시 범위 비율 | Interval | 예시 |
|-------|---|---|---|
| 0 | 50%~100% | 15분 | 12~24시간 |
| 1 | 10%~50% | 1분 | 2~12시간 |
| 2 | 2%~10% | 10초 | 30분~2시간 |
| 3 | 0%~2% | 1초 | 0~30분 |

**핵심 설계 결정**:

- SWR (캐싱 + 자동 재검증)
- uPlot (고성능 차트)

### 3.5 Act (개선) 단계

**발견된 이슈 및 해결**:

#### H-01: Zoom Threshold Logic (한 단계씩 vs 다단계 점프)

**원인**: Design 문서의 Zoom Level 범위 (Level 0: 50%~100%, Level 1: 10%~50%) 를 다단계 직접 점프로 해석

**해결**:
```typescript
// 수정 전: 다단계 점프 가능 (0% → Level 3 직접)
if (zoomRatio < 0.10) {
  targetIndex = 3;  // 3단계 건너뜀 ❌
}

// 수정 후: 한 단계씩만 (0 → 1 → 2 → 3)
if (zoomRatio < 0.50 && currentIndex < levels.length - 1) {
  targetIndex = currentIndex + 1;  // +1만 ✅
}
```

**파일**: `apps/web/src/lib/chart-utils.ts:44-55`

**효과**:
- ✅ 15m → 1m → 10s → 1s 순서 보장
- ✅ 2단계 이상 건너뛰기 완전 차단
- ✅ 역방향 자동 전환 제거
- ✅ 사용자 요구사항 충족 ("한 단계씩만")

#### H-02: Mock Data Field Name Mismatch

**문제**: Mock data가 `current`/`prev` 사용하지만 Design API는 `power`/`air` 명시

```typescript
// 수정 전
data.push({

  current,  // ❌ Design과 불일치
  prev,
});

// 수정 후
data.push({
  time,
  power,    // ✅ Design 준수
  prevPower,
});
```

**파일**: `apps/web/src/services/mock/range-data.ts:51-58`

**효과**: Backend API 연동 시 field name mismatch 방지

#### M-01: API Error Toast Notification

**문제**: API 실패 시 콘솔에만 기록, 사용자에게 피드백 없음

**해결**: `useDynamicResolution` hook의 `onError` 핸들러에 alert/toast 추가

```typescript
onError: (err) => {
  console.error('[useDynamicResolution] API Error:', err);
  alert(`데이터 로드 실패. ${currentInterval} 데이터를 유지합니다.`);  // ✅
},
```

**파일**: `apps/web/src/hooks/useDynamicResolution.ts:62-65`

**효과**: 사용자에게 명확한 에러 피드백 제공

**문제**: Design 섹션 6.2에 정의된 ClientError 타입이 미구현

interface ClientError {

#### M-04: Air Chart Zoom Independence

**원래 설계**: Power chart와 Air chart 모두 독립적으로 zoom 반응

**구현 현황**: Power chart만 onZoomChange 호출, Air chart는 `undefined`

```typescript
// Power chart
<TrendChart onZoomChange={(ratio) => dynamicPowerResolution.handleZoom(ratio)} />

// Air chart
<TrendChart onZoomChange={undefined} />  // Air는 power 따라감
```

**결정**: 의도적 설계 변경 (conflict 방지)
- Power chart zoom → 두 차트 모두 interval 변경
- Air chart zoom → 무시 (power 따라감)
- 두 차트가 서로 다른 level에 있는 것을 방지

**파일**: `apps/web/src/pages/monitoring/MON002LineDetail.tsx:267`

**효과**: 차트 간 동기화 명확성 ✅

## 10. 부록: 기술 상세

### 10.1 Zoom Level 계산 알고리즘

```typescript
// Design vs Implementation 비교
// Design: 범위 비율로 직접 레벨 결정 (0.5=L0, 0.3=L1, 0.05=L2, 0.01=L3)
// Impl: 현재 레벨 기반 단계별 상승 (L0 → L1 → L2 → L3)

function getIntervalForZoomRatio(
  zoomRatio: number,
  currentIndex: number,
  initialInterval: Interval
): Interval {
  const levels =
    initialInterval === "15m"
      ? ["15m", "1m", "10s", "1s"]
      : ["1m", "10s", "1s"];

  // H-01 fix: 한 단계씩만 상승
  let targetIndex = currentIndex;
  if (zoomRatio < 0.50 && currentIndex < levels.length - 1) {
    targetIndex = currentIndex + 1;  // +1만 ✅
  }

  return levels[targetIndex];
}
```
