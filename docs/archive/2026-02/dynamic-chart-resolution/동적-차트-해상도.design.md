## 1. Overview

### 1.1 Design Goals

1. **Progressive Resolution**: 지도 앱처럼 확대할수록 자동으로 더 세밀한 데이터 표시
2. **사용자 편의성**: 수동 interval 변경 불필요, zoom만으로 자동 전환
3. **성능 최적화**: 캐싱으로 중복 API 요청 방지, 디바운싱으로 깜빡임 제거
4. **유연한 시작점**: 15분 또는 1분부터 시작 선택 가능

### 1.2 Design Principles

- **자동화 우선**: 사용자가 zoom하면 시스템이 자동으로 최적 interval 선택
- **캐시 기반**: 동일 Level + 구간은 재요청 없이 캐시 사용 (SWR)
- **무중단 경험**: 디바운싱으로 연속 확대 시 API 폭주 방지, 부드러운 전환
- **기존 호환성**: TrendChart API 변경 최소화 (props 추가만)

### 2.2 Data Flow

```
1. 사용자: "15분" 선택 → initialInterval = "15m"
   └→ API: interval=15m 데이터 로드 (Level 0)

2. 사용자: 차트 드래그 확대 (50% → 30%)
   ├→ TrendChart: setSelect 훅 감지
   ├→ onZoomChange(0.3) 콜백 호출
   └→ useDynamicResolution:
      ├─ getIntervalForZoomRatio(0.3, "15m") → "1m"
      ├─ currentInterval 변경: "15m" → "1m"
      ├─ 500ms 디바운싱
      └─ fetchRangeData(start, end, "1m") → API 요청
         └→ SWR 캐시 확인 → 없으면 API 호출

3. API 응답 → data 업데이트 → TrendChart 리렌더링 (1분 데이터)

4. 사용자: 재확대 (30% → 5%)
   └→ 2번 과정 반복 → interval "10s"

5. 사용자: 최대 확대 (5% → 1%)
   └→ interval "1s" (Level 3)

6. 사용자: "리셋" 버튼 클릭
   └→ initialInterval로 복귀 ("15m")
```

## 3. Data Model

### 3.1 Entity Definition

기존 설비 데이터 모델 재사용 (신규 Entity 없음)

```typescript
// 기존 DataPoint 타입 활용
interface DataPoint {
  time: string;        // "HH:mm:ss" 형식
  power?: number;      // 전력 (kWh)
  air?: number;        // 에어 (L)
  [key: string]: any;  // 추가 메트릭
}

// Zoom Level 정의
type ZoomLevel = 0 | 1 | 2 | 3;

// Interval 타입
type Interval = "15m" | "1m" | "10s" | "1s";

// API 응답 타입
interface RangeDataResponse {
  data: DataPoint[];
  metadata: {
    interval: Interval;
    totalPoints: number;
    returnedPoints: number;
    downsampled: boolean;
    zoomLevel: ZoomLevel;
  };
}
```

### 3.2 Zoom Level → Interval 매핑

```

│ Level 0    | 50% ~ 100%    | 15분     | 12~24시간      │
│ Level 1    | 10% ~ 50%     | 1분      | 2~12시간       │
│ Level 2    | 2% ~ 10%      | 10초     | 30분~2시간     │
│ Level 3    | 0% ~ 2%       | 1초      | 0~30분         │
└───────────────────────────────────────────────────────────┘
```

## 4. API Specification

### 4.1 Endpoint List

| Method | Path | Description | Auth |
|--------|------|-------------|------|
| GET | /api/facilities/:id/power/range | 구간별 전력 데이터 조회 (interval 자동 선택) | Optional |
| GET | /api/facilities/:id/air/range | 구간별 에어 데이터 조회 (interval 자동 선택) | Optional |

### 4.2 Detailed Specification

#### `GET /api/facilities/:facilityId/power/range`

**Purpose**: 지정된 시간 범위의 전력 데이터를 요청한 interval로 조회

**Path Parameters:**
- `facilityId`: 설비 ID (예: "HNK10-000")

**Query Parameters:**

| Parameter | Type | Required | Description | Example |
|-----------|------|----------|-------------|---------|
| `startTime` | ISO8601 | ✅ | 조회 시작 시각 | `2024-01-01T00:00:00Z` |
| `endTime` | ISO8601 | ✅ | 조회 종료 시각 | `2024-01-01T23:59:59Z` |
| `interval` | Enum | ✅ | 데이터 간격 | `"15m"` \| `"1m"` \| `"10s"` \| `"1s"` |
| `maxPoints` | number | ❌ | 최대 반환 포인트 수 (기본: 무제한) | `10000` |

**Request Example:**
```
GET /api/facilities/HNK10-000/power/range?
  startTime=2024-01-01T08:00:00Z&
  endTime=2024-01-01T16:00:00Z&
  interval=1m&
  maxPoints=10000
```

**Response (200 OK):**
```json
{
  "data": [
    { "time": "08:00:00", "power": 3.4 },
    { "time": "08:01:00", "power": 3.5 },
    { "time": "08:02:00", "power": 3.6 },
    ...
  ],
  "metadata": {
    "interval": "1m",
    "totalPoints": 480,
    "returnedPoints": 480,
    "downsampled": false,
    "zoomLevel": 1
  }
}
```

**Error Responses:**

| Status | Code | Message | Cause | Handling |
|--------|------|---------|-------|----------|
| 400 | INVALID_INTERVAL | Invalid interval parameter | interval이 허용 값 아님 | 클라이언트에서 재요청 |
| 400 | INVALID_TIME_RANGE | End time must be after start time | 시간 범위 오류 | 시간 범위 검증 |
| 404 | FACILITY_NOT_FOUND | Facility not found | 설비 ID 없음 | 404 페이지 표시 |
| 500 | DATABASE_ERROR | Database query failed | DB 조회 실패 | 재시도 또는 에러 토스트 |

**Error Response Format:**
```json
{
  "error": {
    "code": "INVALID_INTERVAL",
    "message": "interval must be one of: 15m, 1m, 10s, 1s",
    "details": {
      "received": "30s",
      "allowed": ["15m", "1m", "10s", "1s"]
    }
  }
}
```

### 4.3 Caching Strategy

**SWR 캐싱 키 구조:**

```typescript
// 캐시 키: Level + 구간별 독립 캐싱
const cacheKey = `/api/facilities/${facilityId}/power/range?
  start=${startTime}&
  end=${endTime}&
  interval=${interval}`;

// SWR 설정
const swrConfig = {
  dedupingInterval: 60000,       // 1분간 중복 요청 무시
  revalidateOnFocus: false,      // 포커스 시 재검증 안 함
  revalidateOnReconnect: false,  // 재연결 시 재검증 안 함
  shouldRetryOnError: true,      // 에러 시 재시도
  errorRetryCount: 3,            // 최대 3회 재시도
};
```

## 6. Error Handling

### 6.1 Error Code Definition

| Code | Message | Cause | Handling |
|------|---------|-------|----------|
| `ZOOM_LOOP_DETECTED` | Zoom 무한 루프 감지 | zoom → API → re-render 순환 | `useRef` 플래그로 차단, 경고 로그 |
| `API_TIMEOUT` | API 응답 시간 초과 (>5s) | 네트워크 지연 또는 DB 과부하 | 이전 Interval 데이터 유지, 토스트 알림 |
| `INVALID_ZOOM_RATIO` | 잘못된 Zoom ratio | 계산 오류 | 기본값(Level 0) 사용 |
| `CACHE_MISS_ERROR` | 캐시 조회 실패 | SWR 내부 오류 | API 직접 호출로 폴백 |

### 6.2 Error Response Format

```typescript
// API 에러
interface ApiError {
  error: {
    code: string;
    message: string;
    details?: Record<string, any>;
  };
}

// 클라이언트 에러
interface ClientError {
  type: "ZOOM_LOOP" | "API_TIMEOUT" | "INVALID_ZOOM_RATIO";
  message: string;
  timestamp: Date;
  context: {
    currentInterval: Interval;
    zoomRatio: number;
  };
}
```

### 6.3 에러 처리 전략

```typescript
// 1. API 실패 시 이전 Interval 데이터 유지
const handleApiError = (error: Error, previousInterval: Interval) => {
  console.error(`[DynamicResolution] API failed:`, error);
  toast.error(`${previousInterval} 데이터를 유지합니다.`);
  // 상태 롤백하지 않음 (기존 데이터 표시)
};

// 2. Zoom 무한 루프 방지
const isZoomingRef = useRef(false);
const handleZoom = (ratio: number) => {
  if (isZoomingRef.current) {
    console.warn("[DynamicResolution] Zoom in progress, skipping");
    return;
  }
  isZoomingRef.current = true;
  // ... zoom 로직
  isZoomingRef.current = false;
};

// 3. 디바운싱으로 과도한 요청 방지
const debouncedHandleZoom = useMemo(
  () => debounce(handleZoom, 500),
  []
);
```

## 7. Security Considerations

- [x] **Input validation**: startTime, endTime, interval 파라미터 검증
- [ ] **Authentication**: 선택 사항 (현재 내부 시스템)
- [ ] **Rate Limiting**: 백엔드에서 1분당 100회 제한 권장
- [x] **HTTPS enforcement**: 프로덕션 환경에서만 필수
- [ ] **SQL Injection 방지**: Prepared Statement 사용

**주의사항**: 이 기능은 읽기 전용이므로 보안 위험 낮음. 단, interval 파라미터 검증 필수.

### 10.1 Naming Conventions

| Target | Rule | Example |
|--------|------|---------|
| Components | PascalCase | `TrendChart`, `MON002LineDetail` |
| Hooks | camelCase + `use` prefix | `useDynamicResolution` |
| Functions | camelCase | `getIntervalForZoomRatio()`, `fetchRangeData()` |
| Types | PascalCase | `Interval`, `ZoomLevel`, `RangeDataResponse` |
| Constants | UPPER_SNAKE_CASE | `ZOOM_DEBOUNCE_MS`, `MAX_CACHE_SIZE` |

### 10.2 Import Order

```typescript
// 1. External libraries
import { useState, useRef, useCallback, useMemo } from 'react';
import useSWR from 'swr';
import { debounce } from 'lodash-es';

// 2. Internal absolute imports
import { TrendChart } from '@/components/charts';
import { fetchRangeData } from '@/services/monitoring';

// 3. Types
import type { Interval, ZoomLevel, DataPoint } from '@/types/chart';

// 4. Utilities
import { getIntervalForZoomRatio } from '@/lib/chart-utils';

// 5. Styles
import styles from './MON002LineDetail.module.css';
```

### 10.3 Environment Variables

| Variable | Purpose | Scope | Example |

| `VITE_USE_MOCK` | Mock/Real API 전환 | Client | `true` / `false` |
| `VITE_API_BASE_URL` | API 베이스 URL | Client | `http://localhost:4000` |

## 12. Performance Considerations

### 12.1 최적화 전략

| 항목 | 전략 | 목표 |
|------|------|------|
| **API 응답 시간** | 백엔드에서 interval별 인덱스 생성 | < 300ms |

| **캐시 적중률** | SWR dedupe 옵션 활성화 | > 90% |

### 12.2 성능 모니터링

```typescript
// Chrome DevTools Performance API 활용
const measureZoomPerformance = (zoomRatio: number, interval: Interval) => {
  performance.mark('zoom-start');
  // ... zoom 로직
  performance.mark('zoom-end');
  performance.measure('zoom-duration', 'zoom-start', 'zoom-end');

  const measure = performance.getEntriesByName('zoom-duration')[0];
  console.log(`[Perf] Zoom to ${interval}: ${measure.duration.toFixed(2)}ms`);
};
```
