## 1. Overview

### 1.1 Purpose

현재 TrendChart는 고정된 다운샘플링(5000포인트)을 사용하여 전체 데이터를 표시합니다. 사용자가 차트를 확대(zoom)할 때도 동일한 해상도로 표시되어 세밀한 분석이 어렵습니다.

**해결하려는 문제:**
- 전체 뷰에서 1일 데이터를 15분 간격으로 조회 시: 96개 포인트만 표시
- 확대 시에도 동일한 간격 유지 → 세밀한 분석 불가
- 지도 앱처럼 확대할수록 더 상세한 데이터가 필요

**목표: 계층적 해상도 시스템 (Progressive Resolution)**

확대 수준(Zoom Level)에 따라 자동으로 데이터 간격(interval)을 조정:

```
┌────────────────────────────────────────────────────┐
│ Level 0 (전체 뷰)    : 15분 간격 데이터             │
│   ↓ 1차 확대 (< 50% 범위)                          │
│ Level 1              : 1분 간격 데이터              │
│   ↓ 2차 확대 (< 10% 범위)                          │
│ Level 2              : 10초 간격 데이터             │
│   ↓ 3차 확대 (< 2% 범위)                           │
│ Level 3 (최대 확대)  : 1초 간격 데이터              │
└────────────────────────────────────────────────────┘
```

**핵심 개념:**
- **자동 Interval 선택**: zoom ratio에 따라 15분 → 1분 → 10초 → 1초 자동 전환
- **레벨별 캐싱**: 각 zoom level의 데이터를 독립적으로 캐시
- **부드러운 전환**: 디바운싱으로 깜빡임 방지

### 1.2 Background

**현재 상황:**
- `MON002LineDetail.tsx`: 전체 날짜 범위의 데이터를 한 번에 로드
- `TrendChart.tsx`: 데이터를 5000포인트로 다운샘플링
- uPlot의 zoom 기능은 있지만 데이터 재요청 없음

**비즈니스 요구:**
- 설비 담당자가 특정 시간대의 전력/에어 소비 패턴을 세밀하게 분석 필요
- 이상 패턴 발견 시 해당 구간을 확대하여 정확한 시점 파악 필요

## 2. Scope

### 2.1 In Scope

- [ ] **초기 Interval 선택 (UI)**: 조회 시 드롭다운에서 15분 또는 1분 선택 가능
  - "15분" 선택 → Level 0 시작 (15분 → 1분 → 10초 → 1초)
  - "1분" 선택 → Level 1 시작 (1분 → 10초 → 1초)
- [x] **Zoom 이벤트 감지**: uPlot의 `setSelect` 훅으로 확대 구간 및 비율 감지
- [ ] **Zoom Level 계산**: 현재 표시된 범위 대비 보이는 범위 비율로 레벨 결정
  - Level 0: 100% ~ 50% → 15분 간격
  - Level 1: 50% ~ 10% → 1분 간격
  - Level 2: 10% ~ 2% → 10초 간격
  - Level 3: < 2% → 1초 간격
- [ ] **Interval 자동 선택**: Zoom Level에 맞는 interval로 API 재요청
- [ ] **레벨별 캐싱**: 각 Level + 구간별로 독립 캐시 (SWR)
- [ ] **Interval 전환 디바운싱**: 500ms 디바운싱으로 깜빡임 방지
- [ ] **로딩 인디케이터**: 데이터 로드 중 상태 표시 (레벨 전환 시)
- [ ] **에러 핸들링**: API 실패 시 이전 레벨 데이터 유지
- [ ] **리셋 기능**: 초기 선택한 Interval로 복귀 (15분 또는 1분)

### 2.2 Out of Scope

- ❌ 실시간 스트리밍 (WebSocket)
- ❌ 서버 사이드 데이터 저장소 변경 (TimescaleDB 최적화는 별도)
- ❌ 차트 외 컴포넌트 수정
- ❌ 히스토리 데이터 마이그레이션

## 4. Success Criteria

### 4.1 Definition of Done

- [ ] **4단계 Zoom Level 동작 확인**:
  - Level 0: 15분 간격 (전체 뷰)
  - Level 1: 1분 간격 (1차 확대)
  - Level 2: 10초 간격 (2차 확대)
  - Level 3: 1초 간격 (최대 확대)
- [ ] Zoom 비율에 따른 Interval 자동 전환 정상 동작
- [ ] 각 Level별 백엔드 API 호출 확인 (Network 탭에서 `interval=15m/1m/10s/1s` 파라미터)
- [ ] 동일 Level + 구간 재요청 시 캐시 적중 (API 호출 없음)
- [ ] 리셋 버튼 클릭 시 Level 0 (15분) 복귀
- [ ] Interval 전환 중 로딩 메시지 표시 ("1분 데이터 로딩 중...")
- [ ] TypeScript 타입 에러 없음
- [ ] ESLint 경고 없음

### 4.2 Quality Criteria

- [ ] 기존 차트 기능 regression 없음 (MON001-006 모든 차트 정상)
- [ ] **4단계 Interval 전환 시 깜빡임 없음** (500ms 디바운싱)
- [ ] **Level 전환 속도**: API 응답 < 300ms, 렌더링 < 200ms (총 500ms 이내)
- [ ] Chrome DevTools Performance 탭에서 60fps 유지
- [ ] **메모리 효율성**: 4개 Level 모두 캐시해도 메모리 증가 < 100MB
- [ ] **캐시 적중률**: 동일 구간 재확대 시 90% 이상 캐시 적중

## 5. Risks and Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| 무한 루프 (zoom → API → re-render → zoom) | High | Medium | `useRef`로 API 요청 중 플래그 관리, zoom 이벤트 디바운싱 |
| 과도한 API 요청 (빠른 연속 확대) | Medium | High | 300ms 디바운싱, SWR dedupe 옵션 활성화 |
| 백엔드 API 미구현 | High | Low | Mock 데이터로 프론트엔드 먼저 구현 후 API 연동 |
| 캐시 메모리 누수 | Medium | Medium | LRU 캐시로 최대 10개 구간만 저장, 컴포넌트 언마운트 시 클리어 |
| 기존 차트 props 호환성 깨짐 | High | Low | 새 props는 optional로 추가, 기본값으로 기존 동작 유지 |

## 6. Architecture Considerations

### 6.1 Project Level Selection

| Level | Characteristics | Recommended For | Selected |
|-------|-----------------|-----------------|:--------:|
| **Starter** | Simple structure (`components/`, `lib/`, `types/`) | Static sites, portfolios, landing pages | ☑ |

## 8. Implementation Plan

### 8.1 Phase 1: API 설계 (Backend와 협의)

```typescript
// 제안 API 스펙 (계층적 해상도 지원)
GET /api/facilities/{facilityId}/power/range
Query:
  - startTime: ISO8601 (2024-01-01T08:00:00Z)
  - endTime: ISO8601 (2024-01-01T16:00:00Z)
  - interval: "1s" | "10s" | "1m" | "15m"  // Zoom Level에 따라 자동 선택
  - maxPoints: number (기본 1000, 최대 제한 없음 - 백엔드에서 다운샘플링)

Response:
{
  "data": [
    { "time": "08:00:00", "power": 3.4, "air": 120.5 },
    { "time": "08:00:01", "power": 3.5, "air": 121.0 },  // 1초 간격 예시
    ...
  ],
  "metadata": {
    "interval": "1s",           // 실제 사용된 interval
    "totalPoints": 28800,       // 원본 데이터 포인트 수 (8시간 × 3600)
    "returnedPoints": 28800,    // 반환된 포인트 수
    "downsampled": false,       // 다운샘플링 여부
    "zoomLevel": 3              // 추론된 Zoom Level (0~3)
  }
}
```

**Zoom Level → Interval 매핑 로직 (프론트엔드)**

```typescript
function getIntervalForZoomRatio(visibleRatio: number): string {
  if (visibleRatio >= 0.5) return "15m";      // Level 0: 50%~100%
  if (visibleRatio >= 0.1) return "1m";       // Level 1: 10%~50%
  if (visibleRatio >= 0.02) return "10s";     // Level 2: 2%~10%
  return "1s";                                 // Level 3: <2%
}

// 예시: 24시간 중 30분 보임 → 30/1440 = 0.021 → Level 2 (10초)
```
